How TPX Permissions Work

TPX uses permission nodes to let server owners customize teleport behavior by rank. You can give different groups more homes, shorter warmups, lower cooldowns, cheaper XP costs, private warp access, and custom teleport effects.

Most TPX permission nodes follow this pattern: 

tpx.<system>.<type>.<value>

Common Types

tpa home spawn back rtp deathpoint warp

Home Limits

Home Permission Nodes

Home permissions control how many homes a player can create. TPX checks the player's permissions and uses the highest home amount it finds, while never going below the default max homes value in the config.

Format tpx.homes.<amount>

Examples

tpx.homes.3
tpx.homes.10
tpx.homes.25
tpx.homes.100
TPX registers home limit permissions from tpx.homes.1 through tpx.homes.1000.

How “Highest Wins” Works

Player Has Result
tpx.homes.5 5 homes
tpx.homes.5 + tpx.homes.10 10 homes
tpx.homes.25 + default config is 3 25 homes

Warp Access

Warp Permission Nodes

Warps can be public, OP-only, or locked behind a numeric permission ID. A warp with permission ID 5 requires the node tpx.warp.5.

Format tpx.warp.<id>

Examples

tpx.warp.1
tpx.warp.2
tpx.warp.10
tpx.warp.100

Example Setup

Warp Permission ID Permission Node Who Gets It?
market none none Everyone
vipmine 2 tpx.warp.2 VIP
staffroom 10 tpx.warp.10 Staff
OP-only warps use the configured OP warp permission level instead of a custom warp node.

Timing Perks

Cooldown Permission Nodes

Cooldown permissions let you reduce command cooldowns for certain ranks. TPX checks from 0 seconds up to the configured scan max and uses the lowest matching value. This means lower cooldown nodes are stronger.

Format tpx.cooldown.<type>.<seconds>

Examples

tpx.cooldown.home.30
tpx.cooldown.home.15
tpx.cooldown.home.5
tpx.cooldown.home.0

Rank Example

Rank Node Result
Member none Uses config cooldown
VIP tpx.cooldown.home.15 15 second /home cooldown
Elite tpx.cooldown.home.5 5 second /home cooldown
Admin tpx.cooldown.home.0 No /home cooldown
Registered cooldown permissions support values from 0 through 3600 seconds.

Teleport Delay

Warmup Permission Nodes

Warmup permissions reduce the countdown before teleporting. Like cooldowns, lower values are better, and 0 means instant teleport.

Format tpx.warmup.<type>.<seconds>

Examples

tpx.warmup.home.5
tpx.warmup.home.2
tpx.warmup.home.1
tpx.warmup.home.0
Registered warmup permissions support values from 0 through 300 seconds.

XP Economy

XP Cost Permission Nodes

XP cost permissions reduce how much XP a teleport command costs. Lower values are better, and 0 makes that teleport type free.

Format tpx.xpcost.<type>.<amount>

Examples

tpx.xpcost.home.25
tpx.xpcost.home.10
tpx.xpcost.home.5
tpx.xpcost.home.0
TPX compares costs as total XP points, not just visual Minecraft levels.

Custom Audio

Sound Permission Nodes

Sound permissions let ranks use different configured sound slots for teleport actions. TPX scans from the highest configured ID down to 1, so higher sound IDs win when a player has multiple matching sound permissions.

Format tpx.sound.<type>.<id>

Examples

tpx.sound.home.1
tpx.sound.home.2
tpx.sound.spawn.10
tpx.sound.warp.25

Sound Types Covered

tpa home spawn back rtp deathpoint warp
The ID points to a sound slot in the TPX sounds config. If a slot is disabled or invalid, TPX falls back to the built-in default sound.

Troubleshooting

Common Questions

Why is my permission not working?

Check spelling, the teleport type, and the number. For example, deathpoint has no underscore in permission nodes.

Which permission wins if a player has multiple?

Homes use the highest home amount. Cooldowns, warmups, and XP costs use the lowest value. Sounds and particles use the highest matching ID.

Why does OP bypass my cooldown, warmup, or XP cost?

TPX has config options for OP permission-level bypasses. Disable those bypass options or lower the OP permission level settings if you want staff to follow normal timing/cost rules.

Why does my sound or particle not play?

Make sure the matching global feature is enabled, the player has not disabled it in /tpx settings, the slot ID exists, and the configured sound or particle ID is valid.
